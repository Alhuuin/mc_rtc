Pre-requisites

To run a controller in ROS using mc_rtc_ticker you need to have build mc_rtc with ROS support. You will also need {% link mc_rtc_ros %}.

We assume roscore is already running or that ROS_MASTER_URI points to an active master.

Runing the display interface

Run the following command to start RViZ along with mc_rtc dedicated panel and the robot model you need:

{% highlight bash %} $ roslaunch mc_rtc_ticker control_display.launch robot:=jvrc1 {% endhighlight %}

Where the robot parameter points to an existing file (without the XML extension) in the `rospack find mc_scenario`/robot folder.

By default, this will load the ground environment (default) but you can override this:

{% highlight bash %} $ roslaunch mc_rtc_ticker control_display.launch robot:=jvrc1 env:=door {% endhighlight %}

Where the env parameter points to an existing file (without the XML extension) in the `rospack find mc_scenario`/env folder.

For more complex scenarios (multiple environment) you will have to create your own launch file and RViZ configuration. You are advised to use the existing ones as examples.

Running the controller

To run the controller, use the following command:

{% highlight bash %} $ rosrun mc_rtc_ticker mc_rtc_ticker {% endhighlight %}

Note that you can restart the controller without restarting the display interface.
